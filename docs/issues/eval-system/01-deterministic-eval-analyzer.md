## What to build

实现一个最小可用的确定性评估分析入口，读取已有评测运行结果和用例定义，在不调用 Resume Agent、不调用任何 LLM 的情况下，产出逐用例基础评估结果、结构化 JSON 和终端摘要。

这个切片的目标是先打通“已保存结果 -> 本地分析 -> 可读结论”的端到端闭环，为后续期望字段、失败分类和门禁层提供稳定输入。

## Acceptance criteria

- [ ] 可以通过 `uv run` 在本地运行分析入口，且不要求 `OPENROUTER_API_KEY`。
- [ ] 输入包含已有评测结果和用例定义，输出包含每条 case 的 ID、描述、执行状态、工具调用、耗时和基础通过/失败状态。
- [ ] 输出结构化 JSON 文件，后续脚本可以稳定读取。
- [ ] 终端输出包含总用例数、成功数、失败数和最基本的失败列表。
- [ ] 为最小分析流程添加单元测试或脚本级测试，覆盖成功结果和执行失败结果。

# 执行计划说明

本目录用于保存任务执行计划，目的是让长任务可以暂停、恢复、审阅和复盘，而不依赖单次对话上下文。

## 什么时候要写计划

满足以下任一条件时，先创建计划再动手：

- 任务预计超过 30 分钟
- 需要跨前后端或跨多个模块协作
- 需要分阶段验证
- 需求会持续多轮迭代

## 目录约定

- `active/`：进行中的任务
- `completed/`：已经完成的任务

## 计划模板

每个计划建议包含以下结构：

```md
# 任务标题

## 背景

## 目标

## 非目标

## 步骤

1. [步骤描述] -> verify: [验证命令或检查点]
2. [步骤描述] -> verify: [验证命令或检查点]
3. [步骤描述] -> verify: [验证命令或检查点]

## 决策日志

## 当前状态

## 下一步
```

## 维护规则

- 计划文件名优先使用英文短横线风格，便于检索
- 有明显阶段变化时，及时更新“当前状态”和“下一步”
- 任务结束后，把文件从 `active/` 移到 `completed/`

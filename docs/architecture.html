Chat Resume 架构可视化

基于当前代码静态探查生成。这个项目是一个 Next.js 前端 + FastAPI 后端的 AI 求职应用，核心能力集中在结构化简历编辑、Resume Agent 流式改写、工具级 diff 确认、模拟面试、语音/数字人和导出链路。

生成日期：2026-05-10 仓库：chat-resume 前端：Next.js 14 / React 18 后端：FastAPI / SQLAlchemy 2 / Pydantic v2 Agent：Deep Agents + LangChain + OpenRouter

代码分层地图

前端页面层 frontend/src/app 是 Next.js App Router。主要入口是 /dashboard、/resumes、/resume/[id]/edit、/interviews、/resume/[id]/interview 和 /resume/print。
前端业务状态 useResumeEditor 管加载、编辑、布局、导出；useResumeAutoSave 管防抖保存；useResumeChatPanel 管聊天历史；useStreamingChat 管 SSE 与工具确认。
前端 API 门面 frontend/src/lib/api.ts 封装 Cookie 请求、简历 CRUD、上传/OCR、导出、面试、数字人和聊天历史。直接流式聊天没有走类方法，而是在 hook 中 fetch。
后端 HTTP 入口 backend/app/main.py 注册鉴权中间件、请求日志、DB 指标、CORS 和 /api 路由前缀；entrypoints/http/router.py 聚合所有业务路由。
后端路由层 entrypoints/http 负责协议适配：把请求 DTO、当前用户、数据库 Session 转交给服务层，异常映射成 HTTP 状态码。
领域服务层 services/domain 管用户、刷新会话、文件和简历；services/interview 管面试 session/turn；services/processing 管解析、OCR、导出。
Agent 编排层 ResumeAgent 装配 prompt、Deep Agents runtime 和工具执行器；AgentHarness 把流式事件落到 agent_sessions 和 agent_events。
工具层 tools/resume 暴露可让 Agent 修改结构化简历的业务工具：更新项目简介、更新/新增/删除亮点；长期记忆交给 Deep Agents 官方 memory 文件机制。
数据层 models 和 state/models.py 定义 users、resumes、chat、agent、interview、refresh session；infra/database.py 用 SQLAlchemy engine/session 管请求生命周期。

main.py 的中间件拦截 /api/resumes、/api/interviews、/api/upload、/api/ai、/api/users、/api/tts、/api/asr、/api/digital-human。

4
刷新会话

refresh_sessions 保存服务端刷新令牌状态，支持轮换、吊销和过期判断。

主流程 2：简历编辑、上传与导出

编辑链路

1
页面加载

resume/[id]/edit/page.tsx 通过 useResumeEditor.fetchResume 拉取 /api/resumes/{id}。

2
本地编辑

各编辑器组件只负责局部表单；updateResumeContent 把 section 写回整份 Resume.content 草稿。

3
自动保存

useResumeAutoSave 用 1500ms 防抖调用 PUT /api/resumes/{id}。发送 Agent 消息前会先强制保存草稿。

4
布局配置

resumeLayoutConfig 同步本地缓存和服务端 layout_config，用于控制模块显隐、顺序、间距和一页适配。

上传/导出链路

1
上传解析

/api/upload/resume 保存临时文件，FileService 提取文本，ResumeParser 调 OpenRouter 结构化，再由 ResumeService.create 入库。

2
JD OCR

/api/upload/jd-ocr 校验图片格式和大小，再交给 JDOcrService 识别 JD 文本。

3
PDF 导出

ExportService.export_to_pdf 会构造前端 /resume/print?data=... URL，并用 Playwright 渲染成 PDF，要求前端服务地址可访问。

4
下载保护

导出文件放在 uploads/exports，下载 URL 带用户、过期时间和签名，由 verify_download_token 校验。

主流程 4：模拟面试与语音/数字人

结构化面试 session /api/interviews 负责创建、列表、详情、记录消息、结束和删除。当前创建时不预生成题目，而是保存岗位、公司、JD、模式、语言、难度等上下文。
面试持久化 InterviewSession 保存会话状态和报告字段；InterviewTurn 保存面试官问题、候选人回答、状态和评价。
数字人供应商切换 DIGITAL_HUMAN_PROVIDER 支持 volcengine、liveavatar、Tavus。Volcengine 分支返回 ready，由 WebSocket 代理实时语音对话。
语音能力 /api/asr 提供实时和一次性识别入口，/api/tts 提供文本转语音和音色列表；实际供应商由 services/voice 封装。

架构风险

工具确认同时依赖内存队列和事件表 活跃流式请求靠 confirmation_manager 的内存 queue 唤醒；持久化恢复靠 agent_events。多进程或服务重启时，要明确恢复路径是否覆盖用户体验。
PDF 导出依赖前端服务可访问 ExportService 用 Playwright 打开 FRONTEND_URL 的打印页。部署时后端到前端的网络、环境变量和浏览器依赖都会影响导出。
结构化简历 JSON 是跨层契约 编辑器、预览、Agent 工具、导出、面试默认值都读写 Resume.content。改 schema 时需要同时覆盖前端类型、后端 schema、工具和测试。
外部 AI/语音供应商较多 OpenRouter、Langfuse、LangSmith、MiniMax、Volcengine、Tavus/LiveAvatar 都在请求链路里。错误分类和降级要保持清楚，否则前端只能显示泛化 500/502。
历史表删除顺序敏感 简历删除需要清理聊天、Agent、面试和优化记录等依赖数据。不要假设所有表都有数据库级 cascade。
路由层仍有一些业务细节 例如 resume_agent.py 同时处理 SSE 协议、观测、权限、session 编排和最终落库。后续变复杂时可以继续把编排抽到应用服务。

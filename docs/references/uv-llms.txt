uv 使用约定参考
================

目的
----
这份参考用于统一本仓库中 Python 依赖、虚拟环境和命令执行方式，避免混用 pip、venv 和其他工具。

仓库规则
--------
1. Python 依赖和虚拟环境统一使用 uv。
2. 后端相关命令默认在 backend/ 目录执行。
3. 需要开发依赖时，优先使用项目声明的 extra，而不是手动逐个安装。

常用命令
--------
- 同步依赖：cd backend && uv sync
- 同步开发依赖：cd backend && uv sync --extra dev
- 运行 pytest：cd backend && uv run pytest
- 运行 Ruff：cd backend && uv run ruff check .
- 运行 mypy：cd backend && uv run mypy app
- 运行脚本：cd backend && uv run python path/to/script.py

执行建议
--------
1. 不要假设全局 Python 环境可用，优先使用 uv run。
2. 新增后端依赖前，先确认是否真的需要，并同步更新项目依赖声明。
3. 如果只是运行单个命令，不要先手动 source venv，直接用 uv run。

任务中常见错误
--------------
- 用 pip install 临时装包，导致环境漂移。
- 在仓库根目录直接跑 Python 命令，绕过 backend 的依赖环境。
- 忘记安装 dev extra，就开始执行 pytest、ruff 或 mypy。

推荐工作流
----------
1. 进入 backend 目录。
2. 必要时执行 uv sync --extra dev。
3. 用 uv run 执行测试、检查或脚本。
4. 修改依赖后，确保依赖声明和验证结果一起提交。

OpenAI / Codex 协作参考
========================

目的
----
这份参考用于帮助智能体在本仓库里更稳定地完成任务，而不是替代官方文档。

仓库内使用原则
--------------
1. 先读 AGENTS.md、ARCHITECTURE.md 和 docs/index.md，再决定要深入哪些文件。
2. 把任务拆成目标、非目标、约束、验证四部分，避免只凭自然语言模糊执行。
3. 中大型任务先写执行计划，再开始实现。
4. 完成每个里程碑后立即运行最小充分验证，不要把所有错误留到最后。
5. 优先把决策、计划和状态写回仓库，而不是依赖单次会话记忆。

适合 Codex 的任务写法
--------------------
- 好任务：修复某个可复现问题，并用测试证明修复成立。
- 好任务：新增一个边界清晰的小能力，并说明不改动哪些地方。
- 好任务：做分阶段改造，每步都附带 verify。
- 差任务：帮我把这里优化一下，但没有目标、边界和验收。

推荐的任务模板
--------------
先阅读 AGENTS.md、ARCHITECTURE.md 和相关入口文件。

目标：
<一句话目标>

非目标：
<明确不做什么>

约束：
- Python 依赖和虚拟环境必须用 uv
- 每个新增模块和函数都要有一句中文注释说明为什么存在
- 修复问题时优先先写测试再修复

验证：
- 后端：cd backend && uv run pytest
- 前端：cd frontend && npm run lint
- 前端：cd frontend && npm run type-check
- 前端：cd frontend && npm run build

执行要求：
- 先输出分步骤计划，每步包含 verify
- 实施过程中持续运行验证
- 失败时先定位和修复
- 最后总结改动、验证结果和残留风险

长任务使用建议
--------------
1. 用 docs/exec-plans/active/ 保存进行中的任务计划。
2. 让计划文件承担状态记录，而不是把状态只留在聊天记录里。
3. 如果任务跨前后端，先确认统一的数据流和边界，再分别落实现。

注意
----
如果官方文档、仓库代码和历史文档三者冲突，优先级通常是：
当前任务要求 > 当前代码行为 > AGENTS / ARCHITECTURE > 历史说明文档。
